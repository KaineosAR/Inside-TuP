# review-project.md

Proceso para revisar un proyecto existente.

---

## Objetivo

Evaluar el estado actual del proyecto, detectar problemas y verificar el cumplimiento de las reglas establecidas.

---

## Flujo de trabajo

### 1. Leer el sistema

Leer:

* system/agents.md
* system/do-not.md
* system/tech-stack.md
* system/dev-environment.md
* system/system-coding-rules.md
* system/system-review-checklist.md

---

### 2. Leer la información del proyecto

Leer:

* project/project.md
* project/status.md
* project/project-coding-rules.md
* project/project-review-checklist.md
* project/project-map.md

---

### 3. Analizar el material proporcionado

Revisar:

- archivos compartidos por el usuario
- fragmentos de código compartidos por el usuario

Identificar:

- errores
- inconsistencias
- riesgos
- oportunidades de mejora

No asumir la existencia de archivos, configuraciones o información que no haya sido proporcionada.
No realizar cambios durante esta etapa.

---

### 4. Ejecutar checklists

Verificar:

* system/system-review-checklist.md
* project/project-review-checklist.md

Registrar todos los problemas encontrados.

---

### 5. Registrar hallazgos

Documentar:

- problema detectado
- ubicación
- posible causa
- posible solución

No corregir automáticamente durante esta etapa.

---

### 6. Generar informe

Presentar:

* problemas encontrados
* ubicación de cada problema
* impacto
* posible solución

No aplicar correcciones automáticamente salvo que se solicite.

---

## Regla

La revisión debe identificar problemas antes de proponer cambios.

Si falta contexto relevante para realizar una revisión confiable, solicitarlo antes de continuar.

No asumir información que no haya sido proporcionada.

No aplicar correcciones durante la etapa de revisión.
